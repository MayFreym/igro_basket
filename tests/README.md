# Тесты корзины

Регрессионная сетка на headless Chrome. **Ставить ничего не нужно** — только Chrome,
Node не требуется.

## Запуск

```bash
bash tests/run.sh
```

Выход `0` — всё зелёное, `1` — есть провал. Печатает сводку и каждую FAIL-строку.
Если Chrome лежит не по стандартному пути:

```bash
CHROME="/путь/к/chrome.exe" bash tests/run.sh
```

## Что внутри

| Набор | Файлы | Что проверяет |
|---|---|---|
| **calc** | `calc.test.html`, `calc.scenarios.js` | Чистые расчёты `js/calc.js` без DOM: итоги/скидки/шкала объёма по 6 статусам, акция от 25 шт., снятая галочка, вкладки уценки и акций, формат денег, маска телефона. 81 проверка. |
| **ui** | `ui.test.js` | Настоящие клики/ввод на реальном `index.html`, чтение живого DOM: смена статуса, счётчик, галочки, вкладки, попап, маска в живом поле, гейт кнопки, сохранение фокуса. 20 проверок. |

## Golden-числа

Эталоны в `calc.scenarios.js` **заморожены** с текущего проверенного кода
(20.07.2026, `rrc` «К оплате» = 20 348 ₽ — сходится с `NOTES.md`). Это
характеризационный тест: он ловит *изменение* поведения. Если правка меняет расчёт
намеренно — обнови golden под новые числа (сними их так же, как в `NOTES.md`,
раздел «Как проверять», или из FAIL-строки: там печатается «ждали / было»).

Строки денег хранятся в ASCII; при сравнении варианты пробела (обычный / nbsp /
narrow-nbsp) и минуса нормализуются, чтобы разница ICU не давала ложных провалов.
Отдельный тест при этом проверяет, что `fmt` ставит именно неразрывный пробел.

## Как устроен ui-набор

`update()` в `js/main.js` откладывает `refresh` на `requestAnimationFrame`, а он в
headless под виртуальным временем не срабатывает. Поэтому `run.sh` генерит копию
`index.html` (`_ui-test.html`, в `.gitignore`), подменяет в ней `rAF` на `setTimeout`
и вставляет `<script src="tests/ui.test.js">` перед `</body>`. **Прод не трогается** —
шим живёт только в копии, которая удаляется сразу после прогона.

Эталон поведения для новых тестов — рабочий оригинал в `_old/` (см. `NOTES.md`,
раздел «Как проверять»).
